Detailed Description

The MotionControlType determines how the OpenPnP MotionPlanner will do its planning and how it will talk to the controller.

◆ EuclideanAxisLimits

org.openpnp.spi.Driver.MotionControlType.EuclideanAxisLimits

Apply axis feed-rate, acceleration and jerk limits multiplied by the proper speed factors. The Euclidean Metric is calculated to allow the machine to run faster in a diagonal. All profile motion control is left to the controller.

◆ Full3rdOrderControl

org.openpnp.spi.Driver.MotionControlType.Full3rdOrderControl

◆ ModeratedConstantAcceleration

org.openpnp.spi.Driver.MotionControlType.ModeratedConstantAcceleration

Apply motion planning assuming a controller constant acceleration motion control but moderate the acceleration and velocity to resemble those of 3rd order control, resulting in a move that takes the same amount of time.

◆ SimpleSCurve

org.openpnp.spi.Driver.MotionControlType.SimpleSCurve

Apply motion planning assuming a controller with simplified S-Curve motion control. Simplified S-Curves have no constant acceleration phase, only jerk phases (e.g. TinyG, Marlin).

◆ Simulated3rdOrderControl

org.openpnp.spi.Driver.MotionControlType.Simulated3rdOrderControl

Apply motion planning assuming a controller constant acceleration motion control but simulating 3rd order control with time step interpolation.

◆ ToolpathFeedRate

org.openpnp.spi.Driver.MotionControlType.ToolpathFeedRate

Apply the nominal driver feed-rate limit multiplied by the speed factor to the tool-path. The driver feed-rate must be specified. No acceleration control is applied.
